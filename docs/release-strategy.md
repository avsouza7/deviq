# Release Strategy

## Objetivo

Definir a estratégia oficial de versionamento, releases e distribuição
da implementação de referência do DevIQ.

## Versionamento

-   SemVer.
-   Conventional Commits.
-   Tags para marcos arquiteturais e funcionais.

## Tipos de Release

-   Alpha
-   Beta
-   Release Candidate (RC)
-   Stable

## Compatibilidade

-   Mudanças incompatíveis exigem incremento de versão major.
-   Correções compatíveis utilizam versões patch.
-   Novas funcionalidades compatíveis utilizam versões minor.

## Distribuição

-   Releases publicadas a partir da branch principal.
-   Artefatos gerados automaticamente pela CI.

## Critérios para Release

-   Testes aprovados.
-   Validação de contratos concluída.
-   Documentação sincronizada.
-   Changelog atualizado.
