# User Journey - Retomar um Projeto

## Objetivo

Descrever a experiência de um desenvolvedor que retorna a um projeto
após um período de ausência e utiliza o DevIQ para recuperar rapidamente
o contexto.

## Cenário

Após dias ou semanas sem trabalhar no projeto, o desenvolvedor executa:

    deviq run

e deseja compreender rapidamente o que aconteceu desde sua última
sessão.

## Jornada

1.  Executar `deviq run`.
2.  O DevIQ identifica as execuções mais recentes.
3.  Consolida as mudanças relevantes.
4.  Apresenta o Executive Summary.
5.  Destaca novas decisões arquiteturais.
6.  Resume os aprendizados.
7.  Exibe a evolução das métricas.
8.  Apresenta recomendações para continuidade.

## Resultado Esperado

O desenvolvedor compreende:

-   o que mudou;
-   quais decisões foram tomadas;
-   quais aprendizados surgiram;
-   quais riscos permanecem;
-   onde deve retomar o trabalho.

## Critérios de Aceitação

-   O contexto é recuperado em poucos minutos.
-   Não é necessário revisar manualmente chats, commits e documentos.
-   Todas as informações são navegáveis até as Evidências de origem.
