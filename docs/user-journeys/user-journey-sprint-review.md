# User Journey - Revisão de Sprint

## Objetivo

Descrever como uma equipe utiliza o DevIQ ao final de uma Sprint para
consolidar o conhecimento produzido e apoiar a retrospectiva.

## Cenário

Ao término da Sprint, a equipe executa `deviq run` sobre as execuções do
período para obter uma visão consolidada dos resultados.

## Jornada

1.  Selecionar o período da Sprint.
2.  Consolidar as execuções relacionadas.
3.  Gerar métricas agregadas.
4.  Identificar decisões arquiteturais recorrentes.
5.  Consolidar aprendizados.
6.  Avaliar a evolução da dívida técnica.
7.  Gerar recomendações para a próxima Sprint.

## Resultado Esperado

A equipe compreende:

-   principais entregas;
-   evolução da qualidade;
-   aprendizados coletivos;
-   riscos remanescentes;
-   prioridades para a próxima Sprint.

## Critérios de Aceitação

-   A análise representa toda a Sprint.
-   Os resultados são rastreáveis até as execuções e evidências.
-   O relatório apoia retrospectivas e planejamento.
