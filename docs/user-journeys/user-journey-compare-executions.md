# User Journey - Comparar Duas Execuções

## Objetivo

Descrever a experiência do desenvolvedor ao comparar duas execuções do
`deviq run` para avaliar a evolução do projeto.

## Cenário

Após executar o DevIQ em dois ciclos distintos de desenvolvimento, o
usuário deseja entender se houve evolução, regressão ou estabilidade.

## Jornada

1.  Selecionar duas execuções.
2.  Validar compatibilidade.
3.  Comparar métricas.
4.  Comparar conhecimento produzido.
5.  Comparar dívida técnica.
6.  Identificar tendências.
7.  Exibir diferenças relevantes.
8.  Gerar recomendações para o próximo ciclo.

## Resultado Esperado

O usuário compreende:

-   evolução da contribuição efetiva;
-   mudanças nas métricas;
-   novos aprendizados;
-   dívida técnica criada e removida;
-   tendências positivas e negativas.

## Critérios de Aceitação

-   A comparação utiliza critérios consistentes.
-   Todas as diferenças são rastreáveis.
-   O resultado orienta decisões para o próximo ciclo.
