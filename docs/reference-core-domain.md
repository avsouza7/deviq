# Core Domain da Implementação de Referência

## Objetivo

Definir o núcleo funcional da implementação de referência do DevIQ, preservando os contratos estabelecidos pelas RFCs e pela arquitetura congelada.

## Componentes

- Entidades de domínio
- Value Objects
- Casos de uso
- Serviços de domínio
- Interfaces (Ports)
- Adaptadores (Adapters)

## Princípios

1. Independência de framework.
2. Arquitetura Hexagonal.
3. Imutabilidade sempre que possível.
4. Contratos explícitos.
5. Testabilidade.

## Fora do escopo

- Integrações específicas.
- Interface gráfica.
- Persistência concreta.
- Providers experimentais.
