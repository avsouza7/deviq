# Canonical Vocabulary Model

## Objetivo

Definir o vocabulário oficial do DevIQ.

Este modelo estabelece os termos canônicos utilizados em toda a especificação,
evitando ambiguidades entre documentação, implementação e relatórios.

## Estrutura

Cada termo deve possuir:

- Vocabulary ID
- Nome
- Definição
- Sinônimos permitidos
- Termos relacionados
- Modelos que utilizam o termo

## Regras

- Um conceito possui apenas uma definição oficial.
- Sinônimos podem existir, mas não substituem o termo canônico.
- Novos termos exigem atualização deste modelo.

## Termos Fundamentais

- Session
- Run
- Evidence
- Collector
- Collection Plan
- Analysis Engine
- Correlation
- Knowledge
- Metric
- Report
- Artifact
- Plugin
- Provider
- Integration
- Capability

## Objetivo Final

Garantir uma linguagem única para todas as especificações e futuras implementações do DevIQ.
