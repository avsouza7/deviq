# AI Runtime Certification

## Objetivo

Definir o processo oficial para certificar um agente ou integração como
**DevIQ Runtime Compatible**.

## Escopo

Aplica-se a agentes de IA, extensões de IDE, CLIs e demais
implementações do DevIQ Runtime.

## Pré-requisitos

-   Declarar a versão da AI Runtime Specification suportada.
-   Utilizar artefatos compatíveis conforme a Compatibility Matrix.
-   Executar a Conformance Suite oficial.
-   Validar os Golden Datasets da versão correspondente.

## Processo de Certificação

1.  Verificar compatibilidade de versões.
2.  Executar os Golden Datasets.
3.  Avaliar os resultados com a Conformance Suite.
4.  Registrar evidências da execução.
5.  Emitir o resultado da certificação.

## Resultado

-   Certified
-   Certified with Restrictions
-   Not Certified

## Critérios

Uma implementação somente pode ser considerada certificada quando todos
os requisitos obrigatórios forem atendidos e os resultados forem
reproduzíveis.

## Renovação

A certificação deve ser revalidada sempre que houver mudança de versão
major da especificação ou do Runtime.
