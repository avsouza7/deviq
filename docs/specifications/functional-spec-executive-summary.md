# Especificação Funcional - Executive Summary

## Objetivo

Definir o comportamento funcional do Executive Summary, principal saída do DevIQ
para consumo pelo desenvolvedor ao final de um ciclo de desenvolvimento.

## Finalidade

Responder, de forma objetiva:

- O que foi realizado?
- Qual foi a contribuição efetiva?
- Quais decisões foram tomadas?
- O que foi aprendido?
- Quais riscos permanecem?
- Quais são os próximos passos?

## Entradas

- Report Model
- Metrics Model
- Knowledge Model
- Insight Model
- Recommendation Model

## Estrutura

O Executive Summary deve conter, no mínimo:

- Contexto da execução
- Effective Contribution
- Principais entregas
- Decisões arquiteturais
- Aprendizados
- Métricas relevantes
- Dívida técnica criada
- Dívida técnica removida
- Próximas ações

## Regras

- Deve ser legível em poucos minutos.
- Toda afirmação deve ser rastreável.
- Não deve exigir leitura do relatório completo.

## Critérios de Aceitação

- Resume fielmente a execução.
- Destaca apenas informações relevantes.
- Serve como ponto de entrada para aprofundamento no Report Model.
