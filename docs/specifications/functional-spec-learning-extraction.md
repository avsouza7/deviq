# Especificação Funcional - Learning Extraction

## Objetivo

Definir como o DevIQ identifica e consolida os aprendizados produzidos durante
uma Sessão de desenvolvimento assistido por IA.

## Finalidade

Transformar descobertas, correções, experimentações e decisões em Aprendizados
explícitos, reutilizáveis e compartilháveis.

## Entradas

- Knowledge Model
- Architectural Decisions
- Prompt Timeline
- Evidence Model
- Correlation Model

## Fluxo Funcional

1. Identificar mudanças de entendimento.
2. Detectar problemas resolvidos.
3. Identificar padrões aprendidos.
4. Consolidar aprendizados repetidos.
5. Classificar o tipo de aprendizado.
6. Publicar o Learning Package.

## Saídas

- Learning Package
- Learning Index
- Aprendizados para Executive Summary

## Tipos de Aprendizado

- Técnico
- Arquitetural
- Processo
- Ferramenta
- IA Assistida

## Regras

- Todo aprendizado deve possuir Evidências.
- Aprendizados duplicados devem ser consolidados.
- O processo deve ser reproduzível.

## Critérios de Aceitação

- O desenvolvedor consegue revisar rapidamente o que foi aprendido.
- Os aprendizados alimentam futuras análises e comparações.
