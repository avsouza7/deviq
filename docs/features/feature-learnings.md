# Especificação Funcional - Feature: Learnings

## Objetivo

Definir a experiência do usuário ao consultar os Aprendizados produzidos
pelo DevIQ após uma Sessão de desenvolvimento assistido por IA.

## Problema

Durante uma sessão de desenvolvimento, diversos aprendizados surgem de
experimentações, correções, falhas, revisões e decisões. Esses
conhecimentos normalmente permanecem dispersos entre conversas, código e
documentação.

O DevIQ deve consolidar esses aprendizados automaticamente.

## Perguntas Respondidas

-   O que foi aprendido durante esta sessão?
-   Quais problemas foram resolvidos?
-   Quais boas práticas surgiram?
-   Quais erros deixaram de ocorrer?
-   O que pode ser reutilizado em sessões futuras?

## Conteúdo Obrigatório

-   Aprendizados identificados
-   Classificação dos aprendizados
-   Contexto em que ocorreram
-   Evidências relacionadas
-   Impacto observado
-   Possíveis reutilizações

## Classificação

-   Técnico
-   Arquitetural
-   Processo
-   Ferramenta
-   IA Assistida

## Navegação

Cada aprendizado deve permitir aprofundamento para:

-   Evidências
-   Prompts
-   Decisões Arquiteturais
-   Código relacionado
-   Commits
-   Insights

## Critérios de Aceitação

-   Os aprendizados representam conhecimento efetivamente produzido.
-   Todo aprendizado possui rastreabilidade.
-   O usuário consegue reutilizar rapidamente o conhecimento produzido
    pela sessão.
