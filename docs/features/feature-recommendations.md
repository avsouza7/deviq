# Especificação Funcional - Feature: Recommendations

## Objetivo

Definir a experiência do usuário ao consultar as Recomendações geradas
pelo DevIQ ao final de uma Sessão de desenvolvimento.

## Problema

Após analisar uma sessão, o desenvolvedor precisa saber quais ações
possuem maior potencial de melhorar o projeto, reduzir riscos e aumentar
a efetividade do próximo ciclo.

## Perguntas Respondidas

-   O que devo fazer agora?
-   Quais melhorias têm maior impacto?
-   Quais problemas devem ser priorizados?
-   Quais ações podem ser adiadas?
-   Qual evidência justifica cada recomendação?

## Conteúdo Obrigatório

-   Lista priorizada de recomendações
-   Impacto esperado
-   Esforço estimado
-   Justificativa
-   Evidências relacionadas
-   Métricas afetadas

## Priorização

-   Crítica
-   Alta
-   Média
-   Baixa

## Navegação

Cada recomendação deve permitir aprofundamento para:

-   Insight
-   Métrica
-   Evidências
-   Dívida técnica
-   Decisão arquitetural

## Critérios de Aceitação

-   Todas as recomendações são acionáveis.
-   Cada recomendação possui justificativa rastreável.
-   O usuário consegue definir facilmente o próximo ciclo de
    desenvolvimento.
