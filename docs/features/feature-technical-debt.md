# Especificação Funcional - Feature: Technical Debt

## Objetivo

Definir a experiência do usuário ao analisar a Dívida Técnica
identificada pelo DevIQ após uma Sessão de desenvolvimento.

## Problema

Ao concluir uma entrega, o desenvolvedor precisa compreender não apenas
o que foi implementado, mas também quais compromissos técnicos foram
criados, reduzidos ou eliminados.

## Perguntas Respondidas

-   Qual dívida técnica foi criada?
-   Qual dívida foi removida?
-   Quais riscos permanecem?
-   Quais áreas do projeto foram mais impactadas?
-   O que deve ser priorizado no próximo ciclo?

## Conteúdo Obrigatório

-   Resumo da dívida técnica
-   Itens criados
-   Itens removidos
-   Itens pendentes
-   Classificação por categoria
-   Impacto estimado
-   Evidências relacionadas

## Categorias

-   Arquitetura
-   Código
-   Testes
-   Documentação
-   Dependências
-   Processo

## Navegação

Cada item deve permitir aprofundamento para:

-   Evidências
-   Commits
-   Arquivos modificados
-   Decisões arquiteturais
-   Recomendações

## Critérios de Aceitação

-   A dívida técnica é apresentada de forma objetiva.
-   Todo item possui evidências rastreáveis.
-   O usuário consegue priorizar ações de redução da dívida técnica.
