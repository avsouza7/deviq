# Caso de Uso - Revisar uma Sprint

## Objetivo

Descrever como uma equipe utiliza o DevIQ para revisar uma Sprint
completa, consolidando múltiplas execuções do `deviq run` em uma visão
única.

## Atores

-   Scrum Master
-   Desenvolvedor
-   Tech Lead
-   DevIQ CLI

## Pré-condições

-   Uma Sprint concluída.
-   Execuções do DevIQ disponíveis para o período.

## Fluxo Principal

1.  O usuário seleciona o período da Sprint.
2.  O DevIQ identifica as execuções correspondentes.
3.  Consolida métricas, conhecimento e aprendizados.
4.  Analisa a evolução da dívida técnica.
5.  Identifica decisões arquiteturais recorrentes.
6.  Gera o Executive Summary da Sprint.
7.  Produz recomendações e o Improvement Plan.

## Fluxos Alternativos

### A1. Execuções ausentes

-   O DevIQ informa quais períodos não possuem dados.
-   A análise é realizada com as execuções disponíveis.

## Pós-condições

-   A Sprint possui um relatório consolidado.
-   Os resultados permanecem rastreáveis até cada execução.

## Critérios de Aceitação

-   A análise representa o trabalho realizado durante a Sprint.
-   Todas as conclusões possuem evidências rastreáveis.
-   O relatório apoia retrospectivas e planejamento da próxima Sprint.
