# Provider Core Rule

## Objetivo

Definir os requisitos mínimos que todo provider do DevIQ deve cumprir.

## Regras

1.  Implementar os contratos oficiais do Framework.
2.  Não alterar o modelo canônico de evidências.
3.  Produzir resultados determinísticos para a mesma entrada.
4.  Informar versão, capacidades e limitações.

## Responsabilidades

-   Adaptar chamadas ao provedor.
-   Mapear respostas para o Evidence Model.
-   Preservar rastreabilidade.

## Princípios

-   Provider Agnostic
-   Contract First
-   Evidence First
