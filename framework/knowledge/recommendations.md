# Recommendations

## Objetivo

Definir o contrato canônico para recomendações derivadas do conhecimento
consolidado pelo DevIQ.

## Fontes

As recomendações devem ser geradas exclusivamente a partir de:

-   Lessons Learned;
-   Best Prompts;
-   Reusable Patterns;
-   Evidências consolidadas.

## Estrutura

Cada recomendação deve conter:

-   Identificador;
-   Objetivo;
-   Justificativa;
-   Evidências de suporte;
-   Prioridade;
-   Contexto de aplicação.

## Regras

1.  Toda recomendação deve ser rastreável às evidências de origem.
2.  Não são permitidas recomendações baseadas em opinião.
3.  Recomendações equivalentes devem produzir o mesmo resultado para a
    mesma entrada.

## Princípios

-   Evidence First
-   Determinismo
-   Reprodutibilidade
