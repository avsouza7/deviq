# Contract Validation

## Objetivo

Definir as regras canônicas para validação dos contratos do DevIQ.

## Escopo

Aplica-se a todos os contratos oficiais do Framework.

## Regras

1.  Todo contrato deve possuir versão explícita.
2.  O contrato deve estar em conformidade com seu schema oficial.
3.  Alterações incompatíveis exigem incremento de versão major.
4.  Contratos inválidos não devem ser consumidos pelo Framework.

## Resultado

-   Válido
-   Válido com avisos
-   Inválido

## Princípios

-   Contract First
-   Determinismo
-   Reprodutibilidade
-   Compatibilidade
