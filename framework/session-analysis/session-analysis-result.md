# Session Analysis Result

## Objetivo

Definir o contrato canônico do resultado produzido pelo processo de
análise de sessão.

## Estrutura

O resultado deve conter:

-   Identificador da sessão.
-   Evidências válidas consolidadas.
-   Evidências rejeitadas com justificativa.
-   Métricas calculadas.
-   Data e versão do modelo utilizado.

## Garantias

-   Reprodutibilidade.
-   Determinismo.
-   Compatibilidade com o Evidence Model.
-   Independência de provider.

## Observações

O contrato representa o resultado lógico da análise e não o formato
específico de persistência ou apresentação.
